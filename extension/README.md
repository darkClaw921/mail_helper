# Mail Helper вЂ” Chrome Extension (MV3)

Sidebar for Gmail and Yandex.Mail that renders LLM-classified important
messages from the Mail Helper backend and receives live updates over
WebSocket.

## Requirements

- Chromium-based browser with Manifest V3 support (Chrome 116+, Edge, Brave, Arc)
- Running Mail Helper backend (see `../webapp`) reachable from the browser

## Load unpacked

1. Start the backend: `cd ../webapp && npm run dev`
2. Open `chrome://extensions` in the browser
3. Enable **Developer mode** (toggle, top right)
4. Click **Load unpacked**
5. Pick the `/extension` folder (the one that contains `manifest.json`)
6. The Mail Helper icon should appear in the toolbar

## Configure

1. Right-click the Mail Helper toolbar icon в†’ **Options**
   (or `chrome://extensions` в†’ Mail Helper в†’ Details в†’ Extension options)
2. Set **Backend URL** вЂ” default `http://localhost:3000`
3. Paste your **API Key** (generated by the backend on first start вЂ”
   see `webapp/server/api/auth.js`; also exposed via
   `GET /api/settings` on backend)
4. Click **Save**
5. Click **Test connection** вЂ” should show `/api/health OK` and
   `/api/messages OK` when the key is correct
6. Click **Reconnect WS** to force the background service worker to
   reopen its WebSocket with the new credentials

## Use

- Open `https://mail.google.com/` or `https://mail.yandex.ru/`
- A 380px sidebar appears on the right
- Tabs: **Important** (default) В· **Unread** В· **All**
- The coloured dot in the header shows WebSocket state
  (green = live, amber = connecting, red = offline)
- Click a message card в†’ backend focuses the corresponding mail tab
  (only meaningful for notifications today вЂ” full mail-tab deep-link
  mapping lands with Phase 8)
- Click **Mark read** в†’ `PATCH /api/messages/:id {is_read:1}` and
  (Phase 8) will sync the `\Seen` flag back to IMAP

## Notifications

- In Options, the **Show desktop notification for every important
  message** checkbox is on by default
- OS-level notifications must also be enabled for Chrome
- Clicking a notification focuses the Gmail/Yandex tab and scrolls the
  matching card into view

## Troubleshooting

- **Sidebar does not appear:** Reload the Gmail/Yandex tab after loading
  the extension. Check the inspector вЂ” `[mh-bg]` logs live in the
  service worker console (`chrome://extensions` в†’ service worker
  в†’ *Inspect*).
- **WS dot stays red:** Open Options в†’ Test Connection. If `/api/health`
  fails, the backend URL is wrong or the backend is down. If
  `/api/messages` fails with 401 вЂ” the API key is wrong.
- **No notifications on important mail:** Check Options в†’ Notifications
  checkbox, and the browser's OS-level notification permission for Chrome.
- **Extension disabled after Chrome restart:** MV3 service workers
  are ephemeral. The `chrome.alarms` keepalive (0.5 min period) rewakes
  the worker and reopens the WS. If WS stays down after wake вЂ” click
  *Reconnect WS* in Options.

## Permissions rationale

| Permission / host | Why |
| --- | --- |
| `storage` | Save `backend_url`, `api_key`, collapse state, notify toggle |
| `notifications` | Show desktop notifications on new important mail |
| `alarms` | Wake up SW every 30s to maintain WS |
| `scripting` | Reserved for future per-message focus scroll in Gmail |
| `*://mail.google.com/*` | Inject sidebar in Gmail |
| `*://mail.yandex.ru/*` | Inject sidebar in Yandex.Mail |
| `http://localhost:3000/*` etc. | Let background SW `fetch`/WS to the local backend |
